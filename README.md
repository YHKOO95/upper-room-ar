# 2026 여호수아 비상수련회 AR 컨텐츠

모바일 웹에서 카메라로 숨겨진 이미지 타겟 4개를 비추면 AR 오브젝트가 나타나는 MVP입니다.

## 실행

```bash
npm install
npm run dev
```

모바일 카메라는 HTTPS 환경에서만 안정적으로 동작합니다. 이 프로젝트는 개발 서버도 HTTPS로 실행되도록 설정되어 있습니다.

## 이미지 타겟 준비

앱은 `/public/targets/upper-room-targets.mind` 파일을 찾습니다. 이 파일은 저장소에 포함되어 있지 않으므로, 실제로 사용할 이미지 4장을 MindAR 컴파일러로 변환해야 합니다.

타겟 순서:

1. 빛의 문
2. 말씀의 검
3. 기도의 불씨
4. 비상의 깃발

이미지 타겟을 오브제에 붙이거나 자연스럽게 배치하면, 사용자가 카메라로 해당 위치를 비췄을 때 각 AR 콘텐츠가 나타납니다.

## 구현 구조

- `index.html`: AR 씬과 4개의 이미지 타겟별 3D 오브젝트
- `src/main.js`: 카메라 시작, 타겟 발견 이벤트, 발견 상태 저장
- `src/styles.css`: 모바일 우선 UI와 스캐닝 오버레이
- `public/targets`: MindAR 이미지 타겟 파일 위치
- `public/models`: 실제 GLB/GLTF 모델을 넣는 위치

## 운영 체크리스트

- iOS Safari와 Android Chrome에서 모두 테스트
- 실내 조명과 실제 오브제 위치에서 타겟 인식률 확인
- 이미지 타겟은 반사, 흔들림, 반복 패턴이 적은 디자인으로 제작
- 3D 모델 용량은 모바일 네트워크 기준으로 최적화
